# 代码静态检查

## 一、golangci-lint

* https://golangci-lint.run/

### 1、安装

* https://golangci-lint.run/usage/install/

```plain
brew install golangci-lint
brew upgrade golangci-lint
```

### 2、使用

* https://golangci-lint.run/usage/quick-start/

```plain
golangci-lint run    //相当于运行 golangci-lint run ./...

golangci-lint run dir1 dir2/... dir3/file1.go    //指定文件夹或文件
```

golangci-lint 默认不递归分析目录，如果想递归分析它们，请将/...附加到它们的路径中。

* Golangci-lint 不需要配置就可以使用，执行`golangci-lint help linters`命令可以看到默认开启的linters以及默认禁止的linters。

* 输入`-E/--enable`启动linter，使用`-D/--disable` 禁用linter

```plain
golangci-lint run --disable-all -E errcheck
```

* 使用`--no-config`不使用配置文件。

```plain
golangci-lint run --no-config  --disable-all -E golint,goimports,gofmt
```

* golangci-lint 可以通过`--fix`选项来进行自动修复。比如你可以使用以下命令来对hello.go中不满足`goimports`,`gofmt` 的地方进行自动修复：

```plain
golangci-lint run --no-config  --disable-all -E golint,goimports,gofmt hello.go --fix
```

请注意，并非所有的问题都可以被自动修复，有些问题可能需要手动修复。建议在执行自动修复之后，再次运行 linter 来确认问题是否已经被解决。 另外，确保你安装了支持自动修复的 linter 插件，因为并非所有的 linter 都支持自动修复功能。比如上述命令中的`golint`。查看linter是否支持自动修复：https://golangci-lint.run/usage/linters/
